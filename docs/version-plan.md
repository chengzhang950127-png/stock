# 版本规划文档

**文档定位**：从 V0.1 到 V0.7 的迭代路径，每版的范围、验收标准、价值解锁。

**核心原则**：MVP 先行，每版都"可演示、可使用、可独立交付"，永远不留半成品状态。

**修订规则**：版本范围调整必须更新本文档。WP 评审时校验本版交付物是否覆盖。

---

## 总体节奏

| 版本 | 核心解锁能力 | 累计开发量 | 累计价值 |
|------|------|--------|------|
| **Phase 0.5** | **货币字段补丁（v1.1 新增）** | **~1.25 周** | **解锁跨市场支持的契约层基础** |
| V0.1 | 单策略端到端 | ~1.5 周 | 验证基础架构跑通 |
| V0.2 | 四策略并行（含 GTAA）+ 业绩对比 | ~3.5 周 | 验证 Champion-Challenger 框架 |
| V0.3 | 港股 + Web 仪表盘 | ~5.5 周 | 可向他人展示，公开访问可用 |
| V0.4 | 投资助手 + 主动推送 | ~7.5 周 | 系统能自主给建议 |
| V0.5 | 自定义策略 + 历史库 + 回捞 | ~9.5 周 | 用户参与策略迭代 |
| V0.6 | AI 服务集成 | ~11 周 | 事件理解和可读叙事 |
| V0.7 | 账户系统 + 实盘 | ~13 周 | 真金白银闭环 |

3-4 个并行 Claude Code 窗口同时干活的话，端到端 3-5 周完成核心系统（Phase 0-4），账户系统再加 1 周。Phase 0.5 由于范围极小（单 WP，约 0.25 天），对总周期实际增量在两位小数级，但**必须串行先于 V0.1**。

## Phase 0.5（约 1.5 小时）— 货币字段补丁（v1.1 新增）

### 目标

把 architecture.md v1.1 §10.4 提出的 `Currency` 枚举强制约束落到 contracts.py + ORM + 增量迁移层，作为 V0.1 启动的硬性前置。这一阶段不做任何业务逻辑，只是**接口契约层补丁**。

### 范围

- 新增 `Currency(str, Enum) {USD, HKD}` 枚举
- `Stock` / `Account` / `Position` / `Trade` 加 `currency: Currency` 必填字段
- 4 个 ORM 表加 `currency` 列 + CHECK 约束
- Alembic 增量迁移 `0002_add_currency`
- 新增 `currency_for_market(market) -> Currency` 工具函数
- 测试：4 个模型 round-trip 加 currency；新增工具函数测试

### 不做

- 不增加 FXRate（V0.6 才需要）
- 不增加 CNY（V1.x 扩 A 股时再加）
- 不改 `Signal` 模型（信号继承 stock 的 currency）
- 不改业务逻辑（消费方还没实现）

### 验收标准

- 全部 26 + 新增测试通过（`pytest tests/ -v`）
- 不变量校验通过（`make verify`）
- 迁移在 SQLite 与 Postgres 双方言通过且可逆（`alembic upgrade head` + `alembic downgrade base`）
- CONTRACTS.md 自动重新生成与仓库版本零 diff

### 涉及 WP

WP-0.5.1（详见 `wbs.md` Phase 0.5 章节）

### 工作流约定

特性分支 `wp-0.5-currency`，单 commit 完成，PR 合入 main 后再启动 V0.1 任何 WP。**这是硬约束，不能跳过**。Phase 0.5 跳过会导致 WP-1.x（数据层）和 WP-2.x（策略层）拿到无 currency 字段的 contracts，下游所有 P&L 计算需要二次返工。

## V0.1 MVP（约 1 周）— 验证基础环路

### 目标

用最小代价证明"数据 → 策略 → 信号 → 业绩"环路是通的。一旦 V0.1 跑出有意义的回测结果，整个项目的技术可行性就被证明了。

### 范围

- US 市场数据接入（YFinance 一个数据源）
- 一个策略：**趋势动量**（最简单、因子最少、效果最稳）
- 向量化回测引擎
- 单个影子账户
- CLI 输出：今日信号 + 持仓 + 累计 P&L vs 标普 500

### 不做

- 港股、其他策略、Web 前端、投资助手、推送、用户定制、登录系统、AI 服务

### 验收标准

```bash
python -m run --strategy momentum --period 2020-2025
```

输出回测报告，包含：
- 累计收益、年化收益、夏普、最大回撤
- 持仓变化曲线
- 信号产生记录
- 与 SPY buy-and-hold 对比

**关键早期止损点**：如果趋势动量策略在最近 1-3 年的回测里跑不赢 SPY，必须停下来重新评估——是因子选错？参数没调对？还是当下市场结构不适合这种策略？这种早期反馈比蒙头干 3 个月再发现问题强 100 倍。

### 涉及 WP

Phase 0 全部 + WP-1.1（美股数据） + WP-1.5 简化版 + WP-2.1（基类 + 因子库）+ WP-2.3（趋势动量）+ WP-2.5（信号工具）+ WP-2.7（回测引擎）+ WP-2.8 简化版（单账户）

## V0.2（约 2 周）— 四策略并行

### 目标

验证 Champion-Challenger 框架的工程实现。把 4 套内置策略的"骨架"全部立起来。

### 新增

- 价值反转策略（Value Reversion）
- 事件驱动策略（calendar-only 版本，**不接 AI**，只用结构化日历数据：财报日、FOMC 日等）
- 指数跟踪策略（GTAA，6 个核心 ETF）
- 4 个独立影子账户
- CLI 输出 4 策略并排业绩对比

### 不做

- 港股、AI 服务、Web、助手、自定义、登录

### 验收标准

CLI 输出 4 策略并列业绩对比表（累计收益、夏普、最大回撤、胜率）。

**GTAA 校准点**：Index Following 的回测结果应该能复现 Faber 论文核心数字——2000-2024 年最大回撤显著低于 SPY 同期、年化收益接近 SPY。如果差太远说明实现有 bug。这是 V0.2 的早期止损点。

**初始资本配置**（v1.1 新增明确）：4 个影子账户每个独立 100,000 USD 起始（参见 `architecture.md §4.3`）。**不做总盘瓜分**——让每个策略在完整资金可用条件下表达最佳行为。回测引擎和持仓管理对所有 4 个账户使用相同的资本约束（最小交易粒度、流动性上限、滑点模型），保证业绩对比的可解释性。

### 涉及 WP

WP-2.2（价值反转）+ WP-2.4 简化版（事件驱动 calendar-only）+ WP-2.10（GTAA，新增）+ WP-2.8 扩展（4 账户）

## V0.3（约 2 周）— 港股 + Web 仪表盘

### 目标

从 CLI 工具升级为可访问的产品。这一版完成后，**系统第一次"对外可用"**——可以分享 URL 让别人看到。

### 新增

- 港股数据接入（含南向资金、AH 溢价计算）
- 4 套策略扩展支持港股资产池
- FastAPI 后端 + 公开路由（`/api/public/*`）
- React 前端公开仪表盘（无登录可见）：
  - 4 策略对比卡片
  - 当日信号清单
  - 业绩曲线图（叠加显示）
- 定时任务调度器（每日盘后自动跑分析）
- 数据完整性检查任务（每日运行，检查空缺）

### 不做

- 助手、推送、自定义、登录系统、私有 API

### 验收标准

- 浏览器打开 dashboard URL 看到当日 4 策略状态
- 定时任务每天自动产出新一天的信号和业绩
- 业绩曲线和指标实时更新

### 涉及 WP

WP-1.2（港股数据）+ WP-4.1 简化版（公开 API）+ WP-4.3（前端公开仪表盘）+ 定时调度

## V0.4（约 2 周）— 投资助手 + 主动推送

### 目标

系统从被动展示升级为主动建议者。

### 新增

- Regime 分类器（纯规则，多维度状态判断）
- 资产配置三层规则引擎
- 策略推荐器（Regime → 策略权重映射）
- 助手胜率档案系统
- 主动推送系统（Email + Telegram + 企业微信 至少接一个）
- Regime 切换主动通知
- 助手当日简报页（仍是模板化文本，V0.6 才接 AI 叙事）
- 助手胜率档案页

### 不做

- 自定义策略、AI 服务、登录系统

### 验收标准

- 仪表盘顶部显示当前 Regime + 概率分布
- 助手简报页能看到推荐资产配置和理由
- 触发条件满足时收到推送
- 胜率档案页展示历史建议和事后验证（前期为空，需要时间积累）

### 涉及 WP

WP-3.1（Regime 分类器）+ WP-3.2（资产配置）+ WP-3.3（策略推荐器）+ WP-3.4（胜率档案）+ WP-4.2（推送系统）

## V0.5（约 2 周）— 自定义策略 + 生命周期

### 目标

用户可以参与策略迭代。这是产品的差异化点。

### 新增

- 自定义策略对象（4 维加权组合：value / momentum / event / index）
- 策略管理 UI：
  - 创建对话框（4 个权重滑块）
  - 实时预览综合得分公式
  - 简单回测预览
- 活跃池上限管理：3-5 内置 + 0-2 自定义
- 策略生命周期管理器
- 历史策略库 UI（列表 + 业绩排序 + 回捞按钮 + 删除按钮）
- 回捞流程（业绩重置）
- 删除流程（二次确认）
- 设置页：活跃策略观察数量配置

### 不做

- AI 服务、AI 代码生成、登录系统（仍是公开访问，但加了"观察身份"概念让用户能识别自己的自定义策略）

### 验收标准

- 用户可在 UI 创建 "60% 价值 + 40% 动量" 这类自定义策略
- 创建后立即加入活跃池跑起来，每天产出信号
- 可归档（业绩快照保留）
- 可回捞（业绩重置，重新出生）
- 可删除（二次确认后物理清除）

### 涉及 WP

WP-2.6 简化版（custom 策略对象）+ WP-2.9（策略生命周期管理器）+ 前端策略管理页

## V0.6（约 1.5 周）— AI 服务集成

### 目标

事件驱动从"日历驱动"升级为"新闻驱动"，助手简报从模板化升级为可读叙事。

### 新增

- LLM Gateway 完整实现（Schema 校验 + 版本固定 + 审计日志 + 降级路径）
- 事件驱动策略接入新闻流和 LLM 解析（**第一个 AI 调用点**）
- 助手简报 LLM 文案生成（**第二个 AI 调用点**）
- 所有 LLM 调用走审计日志
- LLM 失败降级测试（验证 LLM 全挂时系统仍能跑）

### 不做

- AI 代码生成、其他 AI 用法

### 验收标准

- 事件驱动策略能基于实时新闻产生信号（不再只靠日历）
- 助手简报变成自然语言而非模板拼接，但所有数字必须从结构化数据来
- 任何 LLM 失败时能降级到 V0.5 行为，决策不停
- 审计日志完整记录每次 LLM 调用

### 涉及 WP

WP-0.3（LLM Gateway 完整版）+ WP-2.4 完整版（事件驱动接 AI）+ AI 叙事生成模块

## V0.7（约 2 周）— 账户系统 + 实盘

### 目标

从信号工具升级为可托管交易的工具。

### 新增

- 用户认证（JWT + email/password 或 OAuth）
- 私有 API 路由（`/api/private/*`）
- 券商凭证加密存储
- 至少一家券商 API 适配（建议先做 IBKR 或富途）
- 自动 / 手动推送两种执行模式可切换
- 前端登录后视图：
  - 持仓页（当前持仓清单、待执行信号）
  - 实施账户页（实时 P&L、交易记录）
  - 账户设置页（券商凭证、执行模式）

### 验收标准

- 登录用户可上传券商 API
- 选择执行模式：手动推送（系统通知用户操作）或自动交易（系统直接下单）
- 可在持仓页看到实施账户实时持仓和 P&L
- 切换执行模式时已有持仓不受影响

### 涉及 WP

WP-5.1（用户认证）+ WP-5.2（凭证存储）+ WP-5.3（券商适配）+ WP-5.4（执行模式）+ 前端登录后页面

## V1.0+ 远期路线（不进当前规划）

下面这些不进入当前 12 周规划，但记录在案以便未来扩展：

- **AI 代码生成自定义策略**：自然语言 → 沙箱代码生成 → 静态分析 → 入活跃池
- **A 股扩展**：通过 AkShare/Tushare 接入 A 股，新增 ASHR/沪深 300 ETF
- **更复杂的 Regime 分类器**：从规则升级为 ML-based（随机森林或 XGBoost）
- **组合优化器**：风险平价权重、Black-Litterman 等专业组合方法
- **回测可视化深度增强**：归因分析、压力测试、蒙特卡洛模拟
- **移动端 App**：iOS / Android 原生客户端
- **多用户多账户**：从个人工具升级为团队 / SaaS

## 关键判断

### MVP 早期止损点很重要

V0.1 完成后必须诚实评估趋势动量策略实测在最近 1-3 年的回测表现。如果跑不赢 SPY，要么策略需要重新设计，要么这条路本来就不通。投入 1 周搞清楚比投入 3 个月发现问题强。

### V0.3 是产品诞生点

前两版是开发者自用的工具，V0.3 之后系统才"对外可用"。这是个心理里程碑——可以分享给朋友看了。

### V0.5 是差异化点

大部分类似产品停在 V0.3 状态。能让用户参与策略迭代是核心壁垒。

### V0.6 推迟到自定义之后是反直觉但正确的

很多产品上来就堆 AI，但 AI 是锦上添花，框架不稳的话 AI 加得越多越乱。等核心稳了再加 AI，AI 的价值反而能被识别出来。

### V0.7 之前可以一直运行公开版本

前 6 个版本期间产品就在持续累积助手胜率档案、影子账户业绩曲线，到 V0.7 上线实盘时已经有 N 个月的真实跟踪数据。**这是巨大的信任建立资产**。
